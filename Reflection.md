# Project Reflection: Roeh 111

## Decisions I Made

The most significant decision was narrowing the scope from a complex application to a professional service platform. I realized that for an MVP, "less is more" with the execution being high quality. I chose to use **Visual Studio Code** and **GitHub Desktop,** and they provided the best visibility into my code structure and version history. Another key decision was implementing a **synchronized design system;** while it took more time to set up initially, this made styling subpages more efficient and professional.

## What Worked

The **"Ghost Engine"** scroll animations worked exceptionally well. By using GPU-accelerated properties, I was able to create a high-end feel that remains smooth on mobile devices. I am also very satisfied with the **branch coloring**; the Gold for the "Life" section was tricky for contrast, but now meets readability standards while looking professional.

## What I Would Do Differently

If I started over, I would follow a **Mobile-First** approach from the very first line of CSS. Retrofitting some of the desktop layouts for smaller screens required more media queries than I would have liked. I also would have standardized my file naming conventions earlier to keep the GitHub repository even cleaner from the start.

## What I Learned

Building for the web taught me that **consistency is the foundation of trust**. If a button behaves differently on the About page than it does on the Home page, the user loses confidence. I also learned a great deal about **risk assessment;** using a matrix to judge impact versus probability helped decide which features to harden and which to cut. Personally, this project reinforced military values of discipline and unit-like trust—applying those concepts to my code structure made the development process much more manageable.
