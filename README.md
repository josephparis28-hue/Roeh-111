# Roeh 111 – Professional Multi-Branch Service Platform

**Live URL:** [https://josephparis28-hue.github.io/Roeh-111/](https://josephparis28-hue.github.io/Roeh-111/)

## What it is

Roeh 111 is a professional MVP designed to provide structured service offerings across three distinct categories: Digital solutions, Restoration projects, and Life/Leadership coaching. It serves as a centralized hub for local Texas clients to engage with specialized services.

## Why it exists

This project exists to bridge the gap between technical web development and hands-on community services. By offering a synchronized interface, users can navigate between different service "branches" while maintaining a consistent, trustworthy brand identity.

## Evolution: What Changed from P01 to P02

* **Themed Synchronization:** Transitioned from basic HTML/CSS to a robust system using CSS variables to maintain branch-specific coloring—Blue for Digital, Green for Restore, and Gold for Life—across both Light and Dark modes.
* **The "Ghost Engine":** Implemented a custom JavaScript scroll-reveal system to enhance user engagement and visual polish.
* **UX Refinements:** Improved the layout by adding significant white space and margins between the "Pampa" branding and CTA sections to enhance readability and flow.
* **Scoped Reduction:** I initially researched an "all-comprehensive" budget application but pivoted to this service-based landing page to prioritize professional credibility and code stability for this deployment.

## Tools Used

* **Visual Studio Code:** Primary IDE for coding and local testing.
